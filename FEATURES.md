# aiPal — Personal Assistant Features

## 1. Conversational Chat Core
The foundation of the assistant. Multi-turn conversations with context awareness. Supports multiple AI providers (OpenAI, Anthropic, Gemini) via Laravel AI SDK. Streaming responses for real-time feel.

---

## 2. Long-Term Memory
The assistant remembers facts about you across sessions — preferences, projects, relationships, recurring topics. Stored as embeddings in **pgvector** and retrieved via semantic search. You never have to re-explain yourself.

---

## 3. RAG — Retrieval-Augmented Generation
Upload your own documents, notes, or code. The assistant indexes them into **pgvector** and answers questions grounded in your personal knowledge base. "What did the spec say about auth?" pulls the right context automatically.

---

## 4. Voice Input (STT)
Speak to the assistant instead of typing. Audio is captured in the browser and sent to Laravel, which transcribes it using **OpenAI Whisper** via the Laravel AI SDK. Works for chat, notes, and reminders.

---

## 5. Voice Output (TTS)
The assistant can speak responses back. Uses **OpenAI TTS** or **ElevenLabs** via Laravel AI SDK. Optional — toggle per session or per message.

---

## 6. Notes
Create, search, and manage personal notes via chat or voice. Notes are stored with embeddings so you can search semantically ("find my note about the deployment issue") rather than by exact keyword.

---

## 7. Reminders
Set reminders through natural language — "remind me to review the PR tomorrow at 9am". Stored in DB, triggered via **Laravel scheduled jobs**. Pluggable delivery channels:
- **Email** (if SMTP configured)
- **WhatsApp** message
- **Telegram** message
- **Generic webhook** — POST a JSON payload to any URL you define (Slack, Discord, ntfy.sh, custom endpoints, etc.)

Users pick their default channels + can override per reminder.

---

## 8. Task & Todo Management
Manage tasks with natural language. Create, update, complete, and prioritize tasks. Ask "what should I focus on today?" and the assistant reasons over your task list, calendar, and priorities.

---

## 9. Daily Briefing
A morning summary delivered on schedule: open tasks, today's calendar events, unread notifications, recent GitLab activity. Generated by the AI and optionally read aloud via TTS.

---

## 10. Calendar Integration
Read Google Calendar events. Ask "what do I have this week?" or "am I free Thursday afternoon?". The assistant uses calendar context when answering scheduling questions.

---

## 11. GitLab Integration
Connect to self-hosted or cloud GitLab via API token. Features:
- Summarize open MRs assigned to you
- "What did I commit this week?"
- Create issues via natural language
- Get notified about MR comments or pipeline failures

---

## 12. Jira Integration
Connect to Jira Cloud or self-hosted Data Center. Features:
- Query your current sprint with JQL
- Create and update issues
- Transition issue status ("move AUTH-42 to In Review")
- Link GitLab MRs to Jira tickets

---

## 13. Email Triage (Gmail / IMAP)
Read and summarize your inbox. The assistant surfaces important emails, drafts replies, and helps you reach inbox zero. Works with Gmail API or any IMAP server for self-hosted setups.

---

## 14. Knowledge Base / Personal Wiki
A structured store of your notes, documents, bookmarks, and meeting notes. All indexed into pgvector. The assistant can answer questions over it, suggest related entries, and keep it organized.

---

## 15. Meeting Notes → Action Items
Paste or record meeting notes. The assistant extracts action items, decisions, and follow-ups, saves them as tasks, and optionally creates Jira tickets.

---

## 16. Code Review Assistant
Paste a diff or point to a GitLab MR. The assistant reviews it for bugs, security issues, and code style, returning structured feedback. Useful for solo devs or pre-review sanity checks.

---

## 17. Terminal Command Helper
Explain or suggest shell commands. "How do I find all files modified in the last 24 hours?" returns the command with explanation. Optionally integrates with a local agent that can execute commands safely.

---

## 18. Pluggable Tool System
Every capability above is a registered **AI tool**. New integrations can be added by implementing a tool class and registering it — no core changes needed. Makes the project extensible for contributors.

---

## 19. Multi-Provider AI Support
Switch between OpenAI, Anthropic, Gemini, Groq, Mistral, and others via Laravel AI SDK. Configure a default provider and fallback for reliability. Different tools can use different models (e.g., Whisper for STT, GPT-4o for chat).

---

## 20. Web UI
A clean, minimal chat interface built with Blade + Alpine.js or Vue. Includes:
- Chat window with streaming
- Voice record button
- Notes and reminders sidebar
- Settings panel for provider/integration config

---

## 21. REST API
Every feature is accessible via a versioned REST API (`/api/v1/`). Authenticated via personal access tokens. Allows third-party clients, CLI wrappers, or mobile apps to be built on top.

---

## 22. Self-Hostable with Docker
Ships with a `docker-compose.yml` that includes:
- Laravel app
- PostgreSQL + pgvector extension
- Redis (queues, cache)
- Nginx

One command to get running. Designed for developers who want data privacy and control.

**Target deployment environments:**
- **VPS** (DigitalOcean, Hetzner, Linode, etc.) — recommended for always-on access
- **Local laptop** — for development or personal offline use
- **Raspberry Pi** (ARM64) — Docker images must be multi-arch (`linux/amd64` + `linux/arm64`); minimum Pi 4 with 4GB RAM recommended

All environments use the same `docker-compose.yml`. No cloud dependency — fully air-gapped capable.

---

## 23. Assistant Persona — Soul & Identity
On first run, an onboarding wizard asks you a series of questions to shape your assistant's identity:
- What should I call myself?
- How should I communicate? (formal / casual / witty / direct)
- What's my backstory or role? (e.g. "you are a senior engineer who loves productivity")
- How do I look? (describe a style, vibe, or character)

From your answers the system generates:
- **Soul file** — a structured personality prompt injected into every conversation (tone, values, communication style, backstory)
- **Cartoon avatar** — generated via an image model (DALL-E 3 or Stable Diffusion) based on your description; used in the web UI and WhatsApp profile
- **Name & display identity** — shown across all interfaces

The persona is stored as a portable JSON file that can be **exported and imported** — move your assistant to a new server, share it, or back it up. Regenerating any part (soul, avatar) is always available from settings.

---

## 25. WhatsApp Bot
Interact with the assistant directly from WhatsApp. Send messages, set reminders, query your notes, ask questions, and get AI responses — all without opening the web UI. Built on the **WhatsApp Business API** (via Twilio or Meta Cloud API). Incoming messages are routed through a webhook to Laravel, processed by the AI, and replied to via the API. Supports the full chat core including long-term memory and tool use.

---

## 26. Telegram Bot
Same capability as the WhatsApp bot, but via **Telegram Bot API**. Usually simpler to set up (just a bot token from @BotFather) and free. Supports text, voice notes (routed through STT), and images. Great for users who don't have WhatsApp Business access.
